## P0 — mid-string interpolation drift (Marc-flagged "DE vs PL" pattern)

Ten EN keys carry two interpolation tokens. The FR/ES/IT/PL bundles
translated the **connector only** (`of` → `de` / `di` / `z`) but **dropped the
adjoining ASCII space**, so every `pageInfo`-shaped surface renders as
`Page 2 de10` instead of `Page 2 de 10`. The same shape ships in five
distinct user paths.

| Key | EN | FR | ES | IT | PL |
| --- | --- | --- | --- | --- | --- |
| `measurements.pageInfo` | `Page {page} of {total}` | `Page {page} de{total}` | `Page {page} de{total}` | `Page {page} di{total}` | `Page {page} z{total}` |
| `mood.pageInfo` | `Page {page} of {total}` | `Page {page} de{total}` | `Page {page} de{total}` | `Page {page} di{total}` | `Page {page} z{total}` |
| `medications.pageInfo` | `Page {page} of {total}` | `Page {page} de{total}` | `Page {page} de{total}` | `Page {page} di{total}` | `Page {page} z{total}` |
| `admin.section.auditLog.pageOf` | `Page {page} of {total}` | `Page {page} de{total}` | `Page {page} de{total}` | `Page {page} di{total}` | `Page {page} z{total}` |
| `admin.showingEntries` | `Showing {count} of {total} auth entries` | `Showing {count} de{total} auth entries` | `Showing {count} de{total} auth entries` | `Showing {count} di{total} auth entries` | `Showing {count} z{total} auth entries` |
| `onboarding.tour.stepOf` | `Step {current} of {total}` | `Step {current} de{total}` | `Step {current} de{total}` | `Step {current} di{total}` | `Step {current} z{total}` |
| `gettingStarted.progress` | `{done} of {total} done` | `{done} de{total} Terminé` | `{done} de{total} Hecho` | `{done} di{total} Fatto` | `{done} z{total} Gotowe` |
| `targets.consistency.daysInRange` | `{count} of {total} in range` | `{count} de{total} in range` | `{count} de{total} in range` | `{count} di{total} in range` | `{count} z{total} in range` |
| `targets.consistency.daysLogged` | `{count} of {total} logged` | `{count} de{total} logged` | `{count} de{total} logged` | `{count} di{total} logged` | `{count} z{total} logged` |
| `targets.summary.weekTitle` | `{met} of {total} targets met this week` | `{met} de{total} targets met this week` | `{met} de{total} targets met this week` | `{met} di{total} targets met this week` | `{met} z{total} targets met this week` |

DE comparison line (correct):
- `gettingStarted.progress` = `{done} von {total} erledigt`
- `admin.showingEntries` = `Zeigt {count} von {total} Auth-Einträgen`

**Fix shape.** Restore the space on both sides of the connector and
finish the partial translation pass:

- FR `Page {page} de{total}` → `Page {page} sur {total}` (or
  `Page {page} de {total}` if Marc prefers "de"; the native French form for
  pagination is "sur").
- ES `Page {page} de{total}` → `Página {page} de {total}`.
- IT `Page {page} di{total}` → `Pagina {page} di {total}`.
- PL `Page {page} z{total}` → `Strona {page} z {total}`.

Apply the same pattern to the other nine rows; nine of ten still ship
the English noun ("Page", "Step", "Showing", "in range", "logged",
"targets met this week") on the four newer locales. Track the partial
translation gap in the P1 section below.

Drift-guard would not catch this — the test asserts presence and
non-empty value but does not assert balanced whitespace around
placeholders. Suggest extending
`src/__tests__/i18n-drift-guard.test.ts` with a regex assertion
`/[A-Za-zÀ-ÿ]\{|\}[A-Za-zÀ-ÿ]/` returning false across the bundle.

## P0 — `dashboard.glp1.*` English leak on FR/ES/IT/PL (R3a B1 scope)

The GLP-1 tile ships live on the dashboard (`src/components/dashboard/glp1-tile.tsx`
lines 278 / 303 / 318 / 324). Four of the eleven `dashboard.glp1.*` keys
were translated to DE only — the FR / ES / IT / PL bundles still hold the
EN string:

| Key | EN / DE / FR / ES / IT / PL |
| --- | --- |
| `dashboard.glp1.lastInjection` | EN `Last injection` · DE `Letzte Injektion` · FR/ES/IT/PL `Last injection` |
| `dashboard.glp1.nextInjection` | EN `Next injection` · DE `Nächste Injektion` · FR/ES/IT/PL `Next injection` |
| `dashboard.glp1.weightDelta` | EN `since start` · DE `seit Beginn` · FR/ES/IT/PL `since start` |
| `dashboard.glp1.inDays` | EN `in {count} days` · DE `in {count} Tagen` · FR/ES/IT/PL `in {count} days` |

The other seven keys in the namespace (`title`, `tabLevel`, `tabWeight`,
`tabsAria`, `rangeStripLabel`, `levelUnavailable`, `weightUnavailable`)
are translated correctly across every locale, so this is a focused
copy-paste regression rather than a missing namespace.

Suggested copy:

| Key | FR | ES | IT | PL |
| --- | --- | --- | --- | --- |
| `lastInjection` | `Dernière injection` | `Última inyección` | `Ultima iniezione` | `Ostatnia iniekcja` |
| `nextInjection` | `Prochaine injection` | `Próxima inyección` | `Prossima iniezione` | `Następna iniekcja` |
| `weightDelta` | `depuis le début` | `desde el inicio` | `dall'inizio` | `od początku` |
| `inDays` | `dans {count} jours` | `en {count} días` | `tra {count} giorni` | `za {count} dni` |

The drift-guard's GLP-1 row would catch a missing key but not an EN-leak.
The Marc-Voice rule plus `cache invalidate on new content` memory both
apply — once the bundles ship, the cached `useTranslations()` instance
on each browser will need a route-change or hard reload to pick up the
new strings.

| `insights.recommendation.*` | 20 | Insight recos |
| `settings.testConnection.*` | 18 | Provider test rows |
| `insights.correlationRow.*` | 16 | Correlation widget |

This is the long tail Marc accepted when shipping FR/ES/IT/PL stubs in
v1.4.25. The locale-integrity test only enforces parity of key **shape**,
not parity of translated value; the drift-guard pins the v1.4.27 new
keys explicitly. Out of v1.4.27 scope as a fix target, but the
`dashboard.glp1.lastInjection` regression came from this same vector
so a future tightening of the drift-guard (e.g. fail when value equals
the EN value AND the EN value contains an alphabetic ASCII letter) would
have caught it.

Decision call for Marc: do we want the drift-guard to start failing on
the wider residue, or scope the v1.4.27 fix to the four GLP-1 rows and
the ten interpolation strings only?

## P2 — `format.dateShort` / `timeShort` / `dateTime` not consumed at runtime

The three keys exist across every locale with the correct native
ordering (DE = `{day}.{month}.{year}`, EN = `{month}/{day}/{year}`,

`src/lib/i18n/__tests__/format-locale-order.test.ts` pins the ordering

actually consume `format.dateShort`, `format.timeShort`, or
`format.dateTime` — every render path still calls
`src/lib/format-locale.ts` (Intl-driven). The test header acknowledges
this ("the actual formatting still routes through `Intl.*`"). Keep the
keys for the PDF / email / CSV downstreams the test comment names,
flag for the v1.4.28 doctor-report work — if those surfaces never wire
up, the keys are dead and the namespace can retire.

## P2 — `app/not-found.tsx` ships hard-coded English copy

`src/app/not-found.tsx` (added in `de0c1633` under R3d MB6) renders three
English literals (`Page not found`, `The page you were looking for ...`,
`Back to dashboard`) without going through `useTranslations()`. The
NotFound page sits **above** the `<I18nProvider>` in the App Router
hierarchy on the same tier as `global-error.tsx`, but unlike
`global-error.tsx` (which carries a `// Keep it static, in English`
docstring justifying the choice) the NotFound page has no equivalent
explanation. A FR / ES / IT / PL visitor who hits a 404 lands on English
copy.

Two viable shapes:
1. Move NotFound into a localised tree (e.g. `src/app/[locale]/not-found.tsx`)
   so it picks up the same provider stack as the live routes.
2. Keep the page bare-bones English (parity with `global-error.tsx`) and
   document the decision inline.

If (2), match the `global-error.tsx` pattern of an inline justification
comment so the next reviewer doesn't open the same ticket.

## P3 — `telegram.helpBody` contains `&lt;` / `&gt;` entities

Every locale's `telegram.helpBody` value includes `&lt;Name&gt;` /
`&lt;name&gt;`. These are **intentional**: Telegram's `parse_mode=HTML`
treats angle brackets as tag markers and requires the entity form for
literal `<` / `>`. The mojibake scan flagged them as the only entity hits
across all six bundles — no fix required. (Documenting so the next
mojibake sweep doesn't re-open the question.)

## Suggested test tightenings (decisions for Marc)

1. **Whitespace-around-placeholder guard.** Extend
   `i18n-drift-guard.test.ts` with a per-locale check that
   `/[A-Za-zÀ-ÿ]\{|\}[A-Za-zÀ-ÿ]/` returns no match. Would have caught
   the ten `pageInfo`-shape strings immediately.
2. **GLP-1 EN-leak guard.** For the `dashboard.glp1.*` namespace
   specifically — every value in FR/ES/IT/PL must differ from its EN
   sibling unless on a small allowlist (brand names, identical
   tokens). The wider 1,664-row residue means this can't go
   bundle-wide without churn, but a narrow per-namespace guard fits
   the drift-guard's existing label shape.
3. **Native-pagination connector test.** A `pageInfo` assertion: the
   FR / ES / IT / PL value contains a space-flanked connector
   matching the native pagination phrase ("sur", "de", "di", "z").
