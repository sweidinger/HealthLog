# v1.4.28 R4 — i18n runtime probe

Scope: 30 commits since `v1.4.27` against the six bundled locales (de, en, fr, es, it, pl).
Pure file-based audit. The locale-integrity test (`src/lib/__tests__/i18n-locale-integrity.test.ts`)
and the drift-guard test (`src/__tests__/i18n-drift-guard.test.ts`) both ran green —
key parity, no-empty values, and the v1.4.27 B6 anchored key sets are intact. The
findings below are therefore content-level, not structural.

## Findings

### Critical — raw-key fallback in production

None. No `useTranslations()` consumer references a key that is missing from any
of the six bundles. All retired keys (`dashboard.glp1.*`, `insights.advisorTitle`,
`insights.aiOverviewTitle`, `insights.weeklyReport.*`, `insights.hero.weeklyReport*`,
`medications.intakeHistory.*`, `medications.inventory.*`) are gone from every
locale and the source tree carries zero consumers for any of them.

### High — EN value leaked into a non-EN locale on a user-facing surface

- **HI-01** — `messages.duplicateTimestamp` ships English copy in fr / es / it / pl.
  Locale: fr, es, it, pl.
  Key: `measurements.duplicateTimestamp` (the prompt called this
  `errors.duplicateTimestamp`; actual namespace is `measurements.*` inside the
  `MeasurementForm` namespace at line 321 of each bundle).
  Consumer: `src/components/measurements/measurement-list.tsx:263` —
  `setEditError(t("measurements.duplicateTimestamp"))`, surfaced on the
  row-edit Sheet whenever the PUT `/api/measurements/[id]` handler now
  returns the v1.4.28 409 with `meta.errorCode = "measurement.duplicate_timestamp"`.
  Value in fr / es / it / pl is the literal EN sentence
  `"A measurement of this type already exists at the selected timestamp."`.
  Recommendation: translate the value in the four affected bundles. This is a
  new v1.4.28 key (commit `538b44f7`) that was added to every locale with the
  same EN string. The four translations are blocker-grade for a release that
  declares itself locale-complete because the banner fires on a routine
  user gesture (re-saving a measurement onto an existing timestamp).

- **HI-02** — `insights.sleep.description` ships English copy in fr / es / it / pl.
  Locale: fr, es, it, pl.
  Key: `insights.sleep.description`.
  Consumers: `src/app/insights/schlaf/page.tsx:39` and `:61` —
  the `<SubPageShell description={…}>` and the empty-state header copy on
  `/insights/schlaf`.
  Value in fr / es / it / pl is the literal EN sentence
  `"Stage breakdown and nightly duration, drawn from Apple Health."`.
  Pre-existing leak (not introduced by v1.4.28) but the v1.4.28 commit
  `8f7cbd49` re-touches the schlaf page so the leak now sits one diff away
  from the maintainer's eye. Recommendation: ship the four translations in
  the v1.4.28 window. Cheap fix, single-line per bundle.

### Medium — EN-leak in non-EN locales on lower-traffic surfaces

- **MD-01** — `insights.sleep.headlineTitle`, `insights.sleep.headlineCaption`,
  `insights.sleep.compositionTitle`, `insights.sleep.compositionSubtitle`,
  `insights.sleep.compositionAriaLabel`, `insights.sleep.stages.{deep,rem,
  core,asleep,awake,inBed,unavailable}`, `insights.sleep.empty.{title,
  description,action}` all carry English values in fr / es / it / pl.
  Locales: fr, es, it, pl.
  Consumers: `src/components/insights/sleep-overview.tsx` (lines 89–140)
  and `src/components/insights/sleep-stage-stacked-bar.tsx` (lines 151–210).
  These render the `<SleepOverview>` + stage-stacked bar on `/insights/schlaf`.
  Pre-existing leak. Same scope-of-fix as HI-02 — once the maintainer touches
  this block, the whole sleep section should be translated in one pass.
  Recommendation: fold into the v1.4.28 fr/es/it/pl sweep alongside HI-01 + HI-02.

- **MD-02** — `insights.confidence` and `insights.cached` ship English copy
  in fr / es / it / pl (`"Confidence"`, `"Cached"`).
  Consumers: every `<InsightStatusCard>` mounted from the six sub-page
  shells (`src/components/insights/insight-status-card.tsx`). User-visible
  on every per-metric sub-page.
  Recommendation: translate. Three-word strings; one-shot fix.

- **MD-03** — `measurements.*` error-form block on the measurement create / edit
  Sheet carries English copy across fr / es / it / pl: `notesPlaceholder`,
  `formReset`, `saveError`, `deleteError`, `invalidValue`,
  `invalidTimestamp`, `loginRequired`, `allTypes`, `filterByType`,
  `measurementCount`, `emptyTitle`, `emptyDescription`, `systolicLabel`,
  `diastolicLabel`, `pulseLabel`, `valueWithUnit`, `glucoseContext*`, plus the
  Apple-Health `type*` labels. v1.4.28's HI-01 (duplicateTimestamp) is the
  newest member of this leak surface, but the whole row should be addressed
  together because the user sees them side by side on the same Sheet.
  Pre-existing.
  Recommendation: out of v1.4.28 R4 scope — flag for a dedicated fr / es / it / pl
  measurement-form translation pass. Tracked separately so HI-01 alone doesn't
  drag the whole block in.

### Low — orphan, placeholder, or dead-letter

- **LO-01** — `insights.healthScore.deltaExplainer.description` (`"Change vs
  last week: {delta}"` + parity translations) is present and translated in
  every bundle but has no consumer.
  The component (`src/components/insights/health-score-delta-explainer.tsx`)
  only reads `trigger`, `title`, and `body`. The fourth key is dead translation
  weight from commit `9a020f21`.
  Recommendation: either wire `description` into the popover body or retire the
  key during the v1.4.28 closure window. Six-locale orphan, no user impact.

- **LO-02** — `insights.coach.window.windowLabel` (`"Window"`) and
  `insights.coach.window.settingsAriaLabel` (`"Coach settings"`) at lines 1059–1060
  remain English in fr / es / it / pl.
  Sibling of the new BK-i18n `lastYear` key that DID get translated in commit
  `75773ca0`. The two neighbour keys were left at EN.
  Surface: coach drawer settings sheet (mobile + desktop). Pre-existing.
  Recommendation: opportunistic fix while the `lastYear` diff is still
  in-window — three locales × two strings.

## Test-suite verdict

- `pnpm vitest run src/__tests__/i18n-drift-guard.test.ts src/lib/__tests__/i18n-locale-integrity.test.ts`
  → 57/57 passing, 172 ms.
- Drift-guard locks `admin.carrier`, `insights.emptyState.*`,
  `notifications.{admin,user}.*`, and `insights.personalRecord.*` to every locale.
  None of the v1.4.28 retirements collide with these locked sets.
- Locale-integrity asserts EN-locale key parity, no-empty values,
  no `TODO/FIXME` placeholders, no duplicate keys at parse time, and the
  no-key-as-value sentinel. The EN-leak findings above all pass because each
  leaked value is a real English sentence, not the key's last segment.

## Summary

- Critical: 0
- High: 2 (`measurements.duplicateTimestamp` × 4 locales is v1.4.28-introduced;
  `insights.sleep.description` × 4 locales is pre-existing but re-touched by the
  v1.4.28 schlaf-page diff).
- Medium: 3 (`insights.sleep.*` body strings, `insights.confidence` + `cached`,
  the measurements-form error block).
- Low: 2 (orphan `deltaExplainer.description`, `coach.window` siblings).
- Coverage holes: fr / es / it / pl for the new `measurements.duplicateTimestamp`
  key. Every other v1.4.28 key set is at full translation parity across the six locales.
- Verdict: **go with caveats**. The release is safe to ship — drift-guard +
  locale-integrity tests are both green and no consumer hits a raw-key fallback.
  Recommend treating HI-01 + HI-02 as v1.4.28-window patches before tagging,
  since `measurements.duplicateTimestamp` will reach the four affected locales
  the first time a non-DE/EN user re-edits a measurement onto an existing
  timestamp, and the schlaf-page header is now one diff away from the
  maintainer's eye.
