## Commit 3 — W7d hardening (`61d156e7`)

Both follow-ups from `.planning/phase-W7d-v1425-dev-server-repair-report.md`
+ `.planning/v1426-backlog.md` §P1-3:

### `safeRequestProp` narrow-catch

`src/lib/api-handler.ts` previously caught every exception from the
request-property read so logging instrumentation never crashed the
wrapped handler. A regression in a header parser or a `null` access
inside `read` would have silently fallen back to the default and
hidden the bug.

The catch is now narrowed to the V8 private-field `TypeError` that
Next 16 raises when the request is a synthetic placeholder on
`force-static` handlers, plus the `Cannot read properties of
undefined/null` shape that surfaces when unit tests invoke the
wrapper with no request argument. Message-shape variants recognised:

- `Cannot read private member #state …` (current V8)
- `Cannot read private field #x …` (alternative V8 wording)
- `Cannot read private name #y …` (Bun / older V8)
- `Cannot read properties of undefined/null …` (modern V8)
- `Cannot read property 'X' of undefined/null` (older V8)

Every other `Error` rethrows. A new `__testables` export exposes the
helpers (`safeRequestProp`, `isTolerableRequestProbeError`) for the
unit-test surface.

The follow-up commit `fcda115` widened the classifier to the
`undefined/null` shape after the initial narrow-catch broke vitest
suites that invoke handlers as `GET()` with no args. The widened
form still rethrows generic TypeErrors (e.g. `undefined is not a
function`, `Cannot read properties of {} (reading 'parse')`), so
real bugs still surface.

### `@source` path resolution

`src/app/globals.css` now imports Tailwind with an explicit
`source("../..")` base. Tailwind v4 auto-detection otherwise walks
up from the CSS file's resolver root, which can land at an unexpected
level depending on how the build tool reaches the stylesheet
(turbopack vs. webpack vs. test runner). Pinning the base mirrors the
cwd-anchored fix the safety-contracts loader landed in `commit dd44a0d`
so every build-tooling integration point reads paths from the same
root.

The `@source not "../../.planning"` + `@source not "../../docs"`
directives stay file-relative (Tailwind resolves them from the
stylesheet, not the source-base option) but their resolved targets
remain the same `<repo-root>/.planning` + `<repo-root>/docs` paths.
Source-base anchoring matters for auto-detection, not for the
explicit `not` exclusions.

### New tests

`src/lib/__tests__/api-handler-safe-request-prop.test.ts` — 11 tests:

- `isTolerableRequestProbeError`: the three private-field message
  wordings classify match; the `Cannot read properties of
  undefined/null` shapes (both modern and older V8 forms) classify
  match; generic TypeErrors, RangeErrors, bare Errors, and non-Error
  values classify non-match.
- `safeRequestProp`: returns the read result on a normal request,
  returns the fallback on a private-field TypeError, rethrows any
  other exception (downstream parser, undefined-is-not-a-function,
  out-of-range). Null-fallback variant covered for header lookups.
