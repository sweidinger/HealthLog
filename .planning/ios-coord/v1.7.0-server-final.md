## Offline sync — server is COMPLETE across all three domains (corrects the earlier "deferred")
The earlier confirmation said mood/intake sync was deferred. That is no longer true:
the server now serves **measurements + mood + intakes** from `GET /api/sync/changes`
over a single multi-domain opaque cursor, with tombstones (soft-delete), `cursorExpired`,
`syncVersion` echo, `/api/sync/state` reporting the window, and `auth.refresh.{revoked,reuse,invalid}`
error codes. Tombstone identity: measurements=`externalId`, mood/intakes=server `id`.
The server side is done and tested; only the iOS consumer for mood/intakes remains on
your roadmap (measurements-first is fine — the contract is frozen and forward-compatible).
