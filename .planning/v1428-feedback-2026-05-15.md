# v1.4.28 — Maintainer Feedback (post-v1.4.27)

Read once: every entry below is something the maintainer hit while walking through the live app at `healthlog.bombeck.io` right after v1.4.27 shipped. Grouped by theme + severity. Marc-Voice English. No PII. Forbidden vocabulary purged.

Severity rubric: **Critical** = live bug, broken feature, can't ship the next release with it; **High** = strong opinion, regression vs. expected behaviour; **Medium** = polish + consistency; **Low** = nice-to-have.

## Recommended v1.4.28 strategy

The list says less scope, more depth than v1.4.27. The big themes that drive sub-passes:

1. **Bug-fix sub-pass** — FB-B1 (workout save), FB-C1 (BD tile numbers), FB-D2 (pulse chart load), FB-D3 (scroll stuck). All Critical, all reproducible, all need real-environment instrumentation. This sub-pass kicks off the release — without it ship is blocked.
2. **Performance sub-pass** — FB-D1 + the suspects called out under Theme D. Measure first (Lighthouse + React profiler + Sentry timing breakdowns), then act. No premature optimisations.
3. **Scope-reduction sub-pass** — FB-A1, FB-A2, FB-E1, FB-E2, FB-J1, FB-J2. Six "remove from code entirely" directives. Clean delete commits, no half-measures.
4. **Consistency sub-pass** — FB-C2, FB-F3, FB-F4, FB-G1, FB-H1-3, FB-K1-2, FB-L1. The "everything looks the same" pass. Audit + apply. Probably the biggest chunk of v1.4.28 by LOC.
5. **Polish sub-pass** — FB-E3 (GLP-1 widget opt-in), FB-F1 + F2 (medications card overflow), FB-I1 (delta tooltip). Self-contained features.
6. **Decision sub-pass** — FB-N (weekly-report retire). One maintainer check, one commit.

The v1.4.27 mobile sweep delivered an architectural shift (ResponsiveSheet, NativeSelect, CoachLaunchProvider). v1.4.28 collects the broken-edges-revealed-by-the-shift plus the consistency directives the maintainer's daily-walk-through surfaces.
